# 04 — 資料品質陷阱(分析前必讀)

> 這份資料是 **AI 生成**的公開資料彙整,做任何分析/引用前須知道以下三個會扭曲結論的陷阱。

## 陷阱 1:`total_funding` 語意不純(混了「募資額」與「公司總規模」)

`total_funding` 欄位**不全是創投募資額**,部分是公司總資產/存款規模:

| 公司 | total_funding | 實際意義 |
|------|--------------|---------|
| Silicon Valley Bank | $209B | **銀行總資產/存款**,不是募資 |
| Wirecard | $28B | 含詐欺虛報的規模 |
| WeWork | $22B | ✅ 確為募資,語意乾淨 |

**分位數**(看出右偏有多嚴重):

| 統計量 | 值 |
|--------|-----|
| min | $0(51 家為 0) |
| 25% | $1.5M |
| 50% | **$5M** |
| 75% | $45M |
| max | **$209B** |
| 平均 | $306M ← 被前 10 大拉爆 |

> **建議**:做「燒錢」分析時,排除或單獨標註 ≥$5B 的 10 家(尤其 SVB);
> 用**中位數($5M)**而非平均($306M)描述「典型」新創。
> ≥$1B 共 88 家,這些大案應視為 outlier 單獨討論。

## 陷阱 2:22 家 `end_year` 在未來(2026–2027)

這些是被 AI **「預判會死/已標記失敗」**而非已實際倒閉,多為 2023–2024 的 AI 新創
(CreatorML、Vango AI、Branch AI、Invaria…)。

> **建議**:做「各年死亡數」時間趨勢時,過濾 `end_year <= 2025`,否則尾端失真。
> 本專案 `03-data-findings.md` 的死亡高峰「2024=203」未含這些未來標記。

## 陷阱 3:8 家百年老牌混入(`start_year < 1990`)

嚴格說不是「新創」,會拉長存活年數尾巴、扭曲年代分布:

| 公司 | start_year | 性質 |
|------|-----------|------|
| Ursus | 1893 | 百年企業 |
| Renown | 1902 | — |
| Onkyo | 1946 | 音響老廠 |
| ITT Tech | 1969 | 連鎖技職學校 |
| VisiCalc | 1979 | 最早倒閉者(end 1984) |
| SVB | 1983 | 銀行 |

> **建議**:分析「新創存活年數」時,可加 `start_year >= 2000` 過濾。

## 其他注意

- **analytics 表只覆蓋 34%(596/1749)**:bottleneck/value-prop/landscape 等
  NLP 欄位的統計只代表這 1/3 子集,不能外推到全庫。
- **`the_loot` 品質參差**:多數是含框架/數字的深度教訓,但少數只是
  「Insight 1: …」的淺層條列。挑素材時優先選長文條目。
- **`landscape_x/y` 是降維座標**:範圍 x∈[-22,38] y∈[-45,33],均值近 0、
  對稱發散 → 典型 UMAP/t-SNE/PCA 2D 嵌入。可直接畫散點,但**軸無物理意義**,
  只代表「題材相似度的相對位置」。
- **資料來源偏向英文+中國大案**:美國 728 / 中國 102,其他地區樣本稀疏,
  地理結論有選樣偏差。
